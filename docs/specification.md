# 幹事AI 仕様書

## 1. サービス概要
サービス名: 幹事AI（仮）
目的: 飲み会幹事が行う以下の作業を簡略化するWebサービス。
- 日程調整
- 集合場所決定
- 店候補提案
- 店投票
1つのURLで飲み会の調整を完結できるサービスとする。

## 2. ユーザー
### 幹事
- イベントを作成する
- URLを参加者に共有する
### 参加者
- URLにアクセスする
- 名前を入力
- 最寄駅を入力
- 日程可否を入力
- 店に投票

## 3. システム構成
フロントエンド: Next.js
バックエンド: Supabase
デプロイ: Vercel
AI: OpenAI API

## 4. URL設計
/                 トップページ
/create           イベント作成ページ
/e/[eventId]      イベントページ

## 5. 画面仕様
### 5.1 トップページ
表示内容: サービス説明, 「イベント作成」ボタン
遷移: /create

### 5.2 イベント作成ページ
入力項目: イベント名, 候補日（複数）
ボタン: イベント作成
処理:
1. eventsテーブルに保存
2. eventId生成
3. /e/[eventId] にリダイレクト

### 5.3 イベントページ
イベントのメインページ
表示内容:
- イベント名
- 参加者入力フォーム
- 参加者一覧
- 日程決定結果
- AI集合場所
- 店候補
- 店投票
- URL共有ボタン

## 6. 参加者入力
入力項目: 名前, 最寄駅, 日程可否
日程可否の入力形式:
○ 参加可能
△ 条件付き
× 不可
送信処理: participantsテーブルに保存

## 7. 日程決定ロジック
候補日ごとにスコアを計算: score = ○ + (△ × 0.5)
最大スコアの日程を決定日程とする
出力: 決定日程, 参加率

## 8. 集合場所AI
入力: 参加者の最寄駅一覧
処理:
1. 繁華街候補リストを用意
2. 各駅からの移動時間を評価
3. 合計移動時間が最小のエリアを選択
繁華街候補: 新宿, 渋谷, 池袋, 東京, 品川, 上野, 恵比寿
出力: 集合エリア

## 9. 公平度表示
参加者ごとの移動時間を表示
公平度スコア: 移動時間の分散を計算しスコア化

## 10. 店候補生成
入力: 集合エリア
AI処理: 飲み会向けの居酒屋を3つ生成
出力: 店A, 店B, 店C

## 11. 店投票
参加者が店に投票（形式: 👍 投票）
集計: 最多票の店を採用

## 12. データベース設計
events
- id
- title
- dates
- created_at

participants
- id
- event_id
- name
- station
- availability

restaurants
- id
- event_id
- name
- votes

## 13. 非機能要件
ログイン: なし
共有: URL共有
対象: スマートフォンブラウザ

## 14. MVP対象外
以下はMVPでは実装しない
- ユーザーアカウント
- 店予約
- 通知
- 履歴
- 幹事ルーレット

## 15. 成功指標
最重要指標: 作成されたイベント数
初期目標: 50イベント
